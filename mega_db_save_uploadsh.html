Description

This script preforms an encrypted backup of a MySql database dumped to bz2 for a given user and sends that backup to Mega.nz account specified and logs the results.

This script by default deletes any backup of the current user database from Mega.nz that is more then 60 days old.

This script by default deletes the backup on the current server once the Mega.nz backup file has been confirmed.


This script was written to work with Virtualmin by default but can easily be configured to work any Linux system running MySql databases.


If there is an error in the backup the script can be configured to send out an email using sendmail. If your using a default install of Virtualmin then sendmail will already be installed.


Script can easily be set to work as a cron job for automated backup.


Configuring mega_db_save_upload.sh

Once you have installed Mega Scripts you will need to edit just a few Fields to get started.

You will need to set the GPG_OWNER field to the "Real Name" you set when you created your GPG Private/Public key-pair.

Change SERVER_NAME to the name of your server.

If you have sendmail installed and configured and you want to send emails on error then set

SEND_EMAIL_ON_ERROR=true
SEND_EMAIL_TO="myemail@example.com"
SEND_EMAIL_FROM="myserver@example.com"

That is really all that you need to get started. Of course make sure you have set up .megarc file of some sort.

Parameters

Param 1

Required: The Unix User to do the database backup for.

Param 2

Required: the name of the database to be backed up to Mega.nz.

Param 3

Optional: The configuration file to use that contain the credentials for the Mega.nz account you want to access.
See Set up .megarc.

Fields

If the field is not listed in the following table then the value should not be altered in the "mega_db_save_upload.sh" script.


Field

Type

Description

ENCRYPT_OUTPUT

true or false

Default is true. If true database is encrypted before sending to Mega.nz. See GnuPG Encrypt and Decrypt

MEGA_ENABLED

true or false

Default is true. If true database is sent to Mega.nz.

DELETE_BZ2_FILE

true or false

Default is true. If true bz2 file of the database dump is deleted after encryption.

MEGA_DELETE_OLD_BACKUPS

true or false

Default is true. If true older backups are deleted from Mega.nz. Date is determined by DAYS_TO_KEEP_BACKUP

DAYS_TO_KEEP_BACKUP

Positive integer

Default is 60. If this value is set to 0 then all backups on Mega.nz will be deleted before the current backup is sent.

true or false

Default is true. If true the local backup is deleted after the backup is confirmed to be on Mega.nz.

SEND_EMAIL_ON_ERROR

true or false

Default is false. If true SEND_EMAIL_TO and SEND_EMAIL_FROM must also be set. Sends an email specified by SEND_EMAIL_TO value if there is an error during the execution of the script. Sendmail must be installed for email to be sent out by this script.

SEND_EMAIL_TO

string

A valid email address to send an email to upon an error. If specifying more then one email then separate each email by comma and space. Example: SEND_EMAIL_TO="myemail1@domain.com, myemail2@domain.com, myemail3@otherdomain.com"

SEND_EMAIL_FROM

string

A valid email address that is used as the From when an email is sent.

string

The GPG public key owner. See GnuPg Encrypt and Decrypt on how to set GPG public key.

string

The name of the current server the script is running on. Used in logs and email.

LOG_NAME

string

The text that is use to separate log entries from different times.

MEGA_DEL_OLD_NAME

string

MEGA_UPLOAD_FILE_NAME

string

Default is "mega_upload_file.sh". The name of the script that is used to upload backups to Mega.nz. This value would only be change if you renamed the "mega_upload_file.sh" script. See mega_upload_file.sh

MEGA_EXIST_FILE_NAME

string

Default is "mega_dir_file_exist.sh". The name of the script that is used to check if a directory or file exist on Mega.nz. This value would only change if you renamed the "mega_dir_file_exist.sh" script. See mega_dir_file_exist.sh

string

Default is "mega_mkdir.sh". The name of the script that is used to make directories on Mega.nz. This value would only change if you renamed the "mega_mkdir.sh" script. See mega_mkdir.sh

string

Default is "/var/log". This the path to where system errors are logged if they occur.

BAK_DIR

string

Default is "/home/${USER}/tmp". Which is the tmp directory of the user passed in via Param 1. This is the directory that the backup of the database is created in.

Setting up Cron

You can set this script to run as a cron job. One entry for each database you want to back up to Mega.nz on a schedule.


To add an entry into cron as root login to the terminal as root user.


# crontab -e


The first time you use cron you may have to choose an editor. I would recommend nano as it is the simplest.

Add your entry at the end of any current cron jobs that exist.

Example Cron entry

The following entry would run once a day at 9:40 PM and upload to the default account set in .megarc.

This entry would backup database "sammy_wordpress" for the user "sammy".

40 21 * * * /bin/bash /usr/local/bin/mega_db_save_upload.sh "sammy" "sammy_wordpress" >/dev/null 2>&1

By Default you would find the log entries in /home/users/sammy/logs/mega_db.log

The file path on Mega.nz would look something like this
"/Root/myserver/backups/sammy/database/2018-06-07-21-40-01_sammy_wordpress.sql.bz2.gpg"




The following entry will run once a day at 1:00 AM and use a second mega account that the credentials stored in the /.megarc_bob

This entry would backup database "bugtracker_bob_db" for the user "bob".

0 1 * * * /bin/bash /usr/local/bin/mega_db_save_upload.sh "bob" "bugtracker_bob_db" ~/.megarc_bob >/dev/null 2>&1

By Default you would find the log entries in /home/users/bob/logs/mega_db.log

The file path on Mega.nz would look something like this
"/Root/myserver/backups/bob/database/2018-06-07-01-00-01_bugtracker_bob_db.sql.bz2.gpg"


See Crontab Generator for an online tool to help generate cron entries.
